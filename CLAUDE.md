# 프로젝트 규칙

힐스테이트 구월아트파크 분양 홍보 상세페이지(Next.js). 아래 규칙은 사용자가 여러 번 직접 지적한 내용을 정리한 것이다. 다음 세션에서도 반드시 지킨다.

## 디자인 방향

- 이 사이트는 한국식 "상세페이지"다. 서구권 SaaS 스타일(작은 좌측 정렬 텍스트, 밋밋한 그리드)로 만들지 않는다.
- 섹션 소개 문구는 가운데 정렬하고 글자 크기를 크게 쓴다.
- 정적인 그리드 나열 대신 스크롤 애니메이션, 무한 롤링 로고 벨트, 카운트업 같은 움직임이 있는 구성을 기본으로 한다. 어워드 수상작이나 구글, Antigravity 같은 빅테크 소개 페이지, 국내 상세페이지(예: isanghan.co.kr, spartaclub.kr) 수준의 완성도를 목표로 한다.
- 참고로 isanghan.co.kr은 WOW.js 스크롤 리빌(fadeInUp)과 jquery.simplyscroll 무한 롤링 로고 벨트, waypoints/counterup 카운트업을 쓴다. 이 프로젝트는 별도 라이브러리를 추가하지 않고 순수 CSS 애니메이션과 IntersectionObserver(Reveal.js 참고)로 같은 효과를 낸다.
- 이미지 처리 방식을 모든 곳에 똑같이 적용하지 않는다. 사진(조감도, 조경, 커뮤니티)과 도면(평면도, 단지 배치도), 로고는 서로 다른 취급이 필요하다.
  - 도면류(평면도, 단지 배치도)는 절대 텍스트 오버레이로 가리지 않는다. 오버레이 대신 이미지 아래에 설명 텍스트를 크게 배치한다. 필요하면 fit="contain"을 써서 잘리지 않게 한다.
  - 사진을 사각 컨테이너에 넣을 때는 실제 이미지 비율을 먼저 확인하고 컨테이너 비율을 크게 벗어나지 않게 맞춘다. 비율 차이가 크면 object-cover 크롭으로 중요한 내용이 잘려나간다.
  - 이미지 하나가 화면 한 번에 다 안 들어올 정도로 커지지 않게 최대 높이를 제한한다.

## 콘텐츠 원칙

- 사용자가 목록 항목 개수를 지정하면(예: "7개") 정확히 그 개수를 지킨다. 임의로 추가하거나 빼지 않는다. 애매하면 결과 보고 시 어떤 기준으로 골랐는지 밝힌다.
- 가격(분양가, 계약금, 중도금 등) 정보는 절대 페이지에 노출하지 않는다(기존 원칙 유지).
- 이 금지는 **이미지 안에 박힌 텍스트에도 똑같이 적용된다.** 사용자가 올린 사진에 "84㎡ 평균 8.52억원" 같은 가격 배너가 들어 있으면 그대로 노출하지 않는다. 새 이미지를 붙일 때마다 가격 문구가 박혀 있는지 눈으로 확인한다.
- **원본 이미지 파일은 절대 수정하지 않는다.** 가격 배너를 지워야 할 때도 원본 픽셀을 덮어쓰지 말고, 컴포넌트에서 CSS 커버(해당 영역 위에 흰 박스를 절대 위치로 얹기)로 비파괴적으로 가린다. (실제 사례: transit-map-guwol/songdo.jpg는 원본 그대로 두고, LivingBenefits 교통 비교 카드에서 좌상단 가격 배너 영역만 흰 박스로 덮음)

## 구현 세부 규칙 (개편으로 확정된 패턴)

- 스크롤 진입 애니메이션은 `components/Reveal.js`(`from="up|left|right"`, delay 지원)로 통일한다. 지그재그 배치에는 좌/우 슬라이드를 방향 맞춰 쓴다.
- 무한 롤링 로고 벨트는 `app/globals.css`의 `.marquee`/`.marquee-track` 순수 CSS로 구현되어 있다. 콘텐츠를 여러 벌 이어붙여 -50% 순환, hover 시 정지, `prefers-reduced-motion`에서 정지된다. 새 라이브러리 추가 금지.
- 숫자 강조 카운트업은 `components/CountUp.js`(IntersectionObserver 기반, 1회 재생)를 쓴다.
- 이미지 컨테이너 비율은 실측 픽셀값을 그대로 `ratio="가로/세로"`로 넣어 잘림을 없앤다(예: 조감도 `2384/1630`, 피트니스 `1536/834`). 21/9 같은 시네마틱 배너는 잘려도 되는 풍경 컷에만 의도적으로 쓴다.

## 한글 문체 규칙

- 번역투와 AI 특유 관용구("결론적으로", "시사하는 바가 크다", 기계적인 "첫째/둘째/셋째" 나열 등)를 쓰지 않는다.
- em dash(—)와 가운뎃점(·)을 장식 기호로 쓰지 않는다. 목록 구분은 쉼표나 줄바꿈, 하이픈(-)을 쓴다.
- 이 저장소에는 humanize-korean 스킬(github.com/epoko77-ai/im-not-ai)이 설치되어 있다(`claude plugin install humanize-korean@im-not-ai`). 분량이 있는 한글 문구를 작성했으면 이 스킬로 검토한다.

## 이미지 자산 관련 참고

- public/images 안의 파일들은 확장자가 .jpg여도 실제로는 PNG 포맷인 경우가 있다. 브라우저 렌더링에는 문제 없다.
- Photo.js는 public/images에 파일이 실제로 있으면 자동으로 그 사진을 쓰고, 없으면 스켈레톤 플레이스홀더를 보여준다(hint prop, fs.existsSync 체크). 새 이미지가 추가되면 별도 코드 수정 없이 자동 반영된다.
