# 디자인 설계: 3분 내 집 마련 준비도 체크

기능 요구사항은 [docs/tasks/self-check.md](../tasks/self-check.md) 참고. 이 문서는 시각 설계만 다룬다.

## 조사 요약

2026년 랜딩 디자인에서 확인한 것 중 이 프로젝트에 실제로 쓸 것과 버릴 것을 갈랐다.

**가져올 것**

- 퀴즈형 퍼널이 정적 폼보다 완료율과 리드 품질이 높다. 문항마다 즉시 반응(마이크로 리워드)이 오면 스크롤 관성이 유지된다. 선택 즉시 다음 문항으로 자동 진행하는 구조를 쓴다.
- 인터랙티브 스크롤. 스크롤에 따라 단계가 조립되듯 펼쳐지는 연출이 국내 프로모션 페이지 표준이 됐다. 진입 티저에 적용한다.
- 모바일 퍼스트. 유입의 대부분이 모바일이고 1초 미만 로딩이 전환에 직접 붙는다. 새 폰트, 새 라이브러리를 추가하지 않는다.

**버릴 것**

- 벤토 그리드. 2026년 표준이라 불리지만 이 기능은 "한 번에 한 문항"이 핵심이라 격자에 얹을 콘텐츠가 아니다. 트렌드라는 이유만으로 쓰면 문항이 카드 더미로 보인다.
- 리퀴드 글래스. 힐스테이트 브랜드 톤(딥 차콜, 골드, 크림슨)과 물성이 충돌하고, 반투명 굴절 위에 한글 본문을 얹으면 가독성이 떨어진다.
- 진행률 퍼센트 숫자 크게 띄우기. 7문항짜리에 "43% 완료"는 과장이고 남은 분량을 오히려 부담스럽게 만든다.

출처는 문서 하단.

## 배치

현재 흐름:

```
Hero → QuickInquiry → LivingBenefits → PropertyDetails → WhyVisit → InfoFAQ → ReservationForm
```

진입 티저를 **WhyVisit 다음, InfoFAQ 앞**에 넣는다.

WhyVisit이 "왜 직접 와야 하는가"로 방문 동기를 만들고 끝난다. 바로 그 자리에서 고객이 품는 다음 질문이 "그런데 내가 지금 살 수 있는 상황인가"다. 이 질문에 답을 주는 자리가 진단 티저다. InfoFAQ보다 앞에 둬야 FAQ가 진단 후 잔여 질문을 받는 역할이 된다.

QuickInquiry 자리(히어로 직후)는 쓰지 않는다. 거기는 이미 초저마찰 전환 접점이고, 7문항짜리를 그 위에 겹치면 두 전환 경로가 서로를 잡아먹는다.

## 토큰

새 팔레트를 만들지 않는다. 브랜드가 이미 고정돼 있고, 진단만 다른 옷을 입으면 별도 서비스처럼 보인다. 기존 `tailwind.config.js` 토큰에서 진단 전용 역할만 배정한다.

| 역할 | 값 | 용도 |
|---|---|---|
| 진단 배경 | `brand` `#1C1917` | 티저 섹션과 설문 화면 전체. 본문 흰 배경 흐름을 끊어 "지금부터 다른 모드"를 알린다 |
| 진행 표시 | `gold` `#B8965A` | 상단 진행 라인, 선택 완료 표시 |
| 선택 상태 | `gold-light` `#D8C39A` | 선택된 보기 테두리와 텍스트 |
| 결과 인장 | `cta` `#5C1C1D` | 결과 화면 도장. 인주 붉은색과 같은 계열이라 물성이 맞는다 |
| 결과 배경 | `ivory` `#F7F4EF` | 설문(어두움) 후 결과에서 밝게 전환. 진단이 끝났다는 신호 |

타이포는 Pretendard 유지. 진단 화면에서만 문항 번호에 `tabular-nums`를 걸고 굵기 대비를 크게 벌린다. 문항은 `text-2xl~3xl font-extrabold`, 보기는 `text-base font-semibold`, 안내는 `text-sm text-white/55`. 세 단계 이상 섞지 않는다.

## 시그니처: 3단 관문 인장

이 페이지가 기억될 하나의 요소.

결과 화면에서 세 항목이 위에서 아래로 순차 점등하며 각각 붉은 인장이 찍힌다.

```
주택 상태        무주택          [印]
대출 상태        확인 필요        [印]
방문 준비        상담 예약 가능    [印]
```

- 인장은 SVG 원형 테두리 + 한자 한 글자(確). 크림슨 `#5C1C1D`, `mix-blend-multiply`로 종이에 스민 느낌.
- 등장은 `scale(1.25) → scale(1)` + opacity, 각 항목 180ms 간격. 회전이나 튕김은 넣지 않는다.
- 왜 인장인가. 분양 상담의 실제 물성은 서류와 날인이다. 계약서에 도장을 찍는 행위가 이 고객이 향하는 종착점이고, 진단 결과에 미리 한 번 찍어보는 것이 이 기능의 정서적 보상이다. 체크마크나 초록 배지로 대체하면 어떤 SaaS 온보딩과도 구분되지 않는다.
- 주의. 인장은 "심사 통과"가 아니라 "확인 완료"다. 유형 C(1주택 이상)에서도 세 개 모두 찍힌다. 다만 가운데 항목 라벨이 "처분 계획 확인 필요"로 바뀐다. 인장의 유무로 합격 불합격을 연상시키면 확정 표현 금지 원칙을 어기게 된다.

보수적인 대안이 필요하면 한자 대신 골드 원형 링에 얇은 체크만 남긴다. 다만 그 경우 시그니처는 사라지고 평범해진다.

## 화면별 설계

### 1. 진입 티저 (랜딩 내 섹션)

`bg-brand` 풀블리드. 좌우 2단이 아니라 가운데 정렬 단일 축으로 간다. 상세페이지 문법대로 소개 문구를 크게, 가운데.

```
┌──────────────────────────────────────────┐
│              (bg-brand, 어두움)            │
│                                          │
│           SELF CHECK  ← gold 라벨          │
│                                          │
│        지금 내 상황에서                     │
│        무엇부터 준비해야 할까요               │
│         ← text-3xl~5xl extrabold          │
│                                          │
│   7개 질문에 답하면 방문 전 확인할 항목을        │
│   정리해 드립니다. 금액은 묻지 않습니다.        │
│                                          │
│      ┌────────┬────────┬────────┐        │
│      │  01    │  02    │  03    │        │
│      │ 주택   │ 대출   │  결과   │        │
│      │ 상태   │ 상태   │  확인   │        │
│      └────────┴────────┴────────┘        │
│        ↑ 스크롤 진입 시 좌→우 순차 점등        │
│                                          │
│        [ 3분 진단 시작하기 ]  ← gold 버튼    │
│                                          │
│     소요 3분 · 소득과 금액은 입력하지 않습니다   │
└──────────────────────────────────────────┘
```

세 칸의 번호(01/02/03)는 장식이 아니다. 실제로 이 순서로 판정이 진행되고, 앞 단계 답이 뒤 단계 해석을 바꾼다. 순서가 정보를 담고 있어서 번호를 쓴다.

기존 `Reveal.js`로 좌측 슬라이드 + delay 70ms 스태거. 새 애니메이션 유틸을 만들지 않는다.

### 2. 설문 화면 (`/check`)

풀스크린, `bg-brand`. 헤더는 로고와 나가기만 남긴 축소판. `FloatingCTA`는 이 경로에서 숨긴다. 하단 고정 CTA와 보기 버튼이 겹치면 마지막 보기를 못 누른다.

```
┌──────────────────────────────────────────┐
│ ▬▬▬▬▬▬▬▬░░░░░░░░░░░░░░  ← 상단 골드 라인 2px │
│ 힐스테이트 구월아트파크            나가기 ✕   │
│                                          │
│                                          │
│        03 / 07   ← tabular-nums, gold    │
│                                          │
│        지금 어떤 형태로                    │
│        거주하고 계신가요                    │
│                                          │
│   ┌────────────────────────────────┐     │
│   │  전세                        →  │     │
│   ├────────────────────────────────┤     │
│   │  월세                        →  │     │
│   ├────────────────────────────────┤     │
│   │  자가                        →  │     │
│   ├────────────────────────────────┤     │
│   │  부모님 댁                    →  │     │
│   └────────────────────────────────┘     │
│                                          │
│              ← 이전 질문                   │
└──────────────────────────────────────────┘
```

- 보기는 카드형 버튼. `rounded-xl border border-white/12 bg-white/[0.06]`, 최소 높이 60px(모바일 터치 타깃).
- 기본 → hover: 배경 `white/[0.10]` + 우측 화살표가 4px 오른쪽으로. focus-visible: 골드 2px 아웃라인. 이 프로젝트는 이미 `lift-card`가 있지만 설문 보기에는 쓰지 않는다. 카드가 떠오르는 모션은 "정보"에 어울리고 "선택"에는 눌리는 반응이 맞다.
- active: `scale(0.985)` + 배경이 `gold/20`으로 한 프레임 번쩍인 뒤 다음 문항으로 넘어간다. 이 200ms가 마이크로 리워드다.
- 문항 전환은 좌우 슬라이드. 다음은 오른쪽에서, 이전은 왼쪽에서. 방향이 진행 방향과 일치해야 한다.
- 진행 라인은 상단 2px 골드. 퍼센트 숫자를 띄우지 않는다. 대신 `03 / 07`을 문항 위에 둔다. 남은 양이 정확히 보이면서 부담스럽지 않다.
- 마지막 8번째 화면에서만 이름, 연락처, 동의를 받는다. 입력 스타일은 `QuickInquiry`의 `.quick-input`을 어두운 배경용으로 변형해 쓴다.

### 3. 결과 화면

`bg-ivory`로 전환. 어두운 설문을 지나 밝은 결과로 나오는 대비 자체가 완료 신호다.

```
┌──────────────────────────────────────────┐
│              (bg-ivory, 밝음)              │
│                                          │
│            홍길동님 확인 결과                │
│                                          │
│   ┌────────────────────────────────┐     │
│   │  주택 상태    무주택        [印] │     │
│   │  대출 상태    확인 필요      [印] │     │
│   │  방문 준비    상담 예약 가능  [印] │     │
│   └────────────────────────────────┘     │
│         ↑ 시그니처. 순차 점등              │
│                                          │
│      기존 대출 상환 계획에 따라              │
│      준비 방법이 달라집니다                  │
│      ← text-2xl~3xl, 가운데                │
│                                          │
│   방문 시 이것만 확인하면 됩니다              │
│   ─────────────────────────────           │
│   ✓ 무주택 확인 자료                       │
│   ✓ 세대 구성 기준                         │
│   ✓ 기존 대출 잔여 조건                     │
│   ✓ 입주 시점 자금 일정                     │
│                                          │
│        [ 홍보관 방문예약 ]                  │
│        [ 전화로 상담받기 ]                  │
│                                          │
│  정확한 공급조건과 금융 가능 여부는            │
│  홍보관 방문과 금융기관 심사를 통해            │
│  확인해 주세요.                            │
└──────────────────────────────────────────┘
```

- 체크리스트는 인장 카드와 다른 물성으로. 흰 배경 카드에 얇은 상단 골드 라인 하나. 두 블록이 같은 스타일이면 시그니처가 묻힌다.
- CTA는 두 개까지. 7번 문항에서 상담 희망을 "예"로 답한 경우 전화상담을 위로 올리고 크림슨 강조를 준다.
- 결과 화면에는 어떤 금액도 없다. 숫자는 문항 번호와 타입 면적(59/74/84㎡)만 등장한다.

## 모션 예산

이 페이지에 넣는 움직임은 네 개다. 그 이상은 넣지 않는다.

1. 티저 3단계 좌→우 순차 점등 (스크롤 진입, 1회)
2. 상단 진행 라인 채움 (문항 이동마다)
3. 보기 선택 시 눌림 + 번쩍임 (200ms)
4. 결과 인장 3연속 점등 (1회)

`prefers-reduced-motion`에서는 전부 정지하고 최종 상태만 보여준다. `globals.css`에 이미 전역 처리가 있으므로 인장은 `animation`이 아니라 `opacity` 전환으로 구현하면 자동으로 걸린다. 인장이 아예 안 보이는 사태를 막으려면 초기 opacity를 0으로 두지 말고 JS로 클래스를 붙이는 방식을 쓴다.

## 하지 말 것

- 로딩 스피너로 "분석 중입니다" 3초 끌기. 계산할 게 없는데 연출로 시간을 끄는 건 고객을 속이는 것이고, 담당자가 말한 "정확하지 않아도 된다"의 취지와도 반대다. 결과는 즉시 보여준다.
- 결과에 점수, 등급, 게이지 표시. A/B/C는 내부 분류이지 고객에게 보여줄 성적이 아니다. 화면에는 유형 코드를 노출하지 않고 문장으로만 안내한다.
- 진단 도중 방문예약 배너 삽입. 완료율을 떨어뜨린다. CTA는 결과 화면에서만.
- 좌측 정렬 소형 텍스트로 문항 배치. 상세페이지 문법에 어긋난다.

## 구현 시 재사용

| 필요 | 기존 자산 |
|---|---|
| 스크롤 진입 | `components/Reveal.js` |
| 입력 스타일 | `QuickInquiry`의 `.quick-input` |
| 전화번호 포맷, 검증 | `lib/phone.js` |
| 색상, 그림자, 애니메이션 | `tailwind.config.js` 기존 토큰 |
| 아이콘 | `components/Icons.js` (인장만 신규 SVG) |
